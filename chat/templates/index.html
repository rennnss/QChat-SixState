⚛️ How Quantum Key Distribution Works

This chat uses the Six-State QKD Protocol to generate encryption keys using quantum mechanics. Unlike classical encryption, any eavesdropping attempt physically disturbs the quantum states, making it detectable.

Here's exactly what happens each time you send a message:

1

🔵 Alice Prepares Qubits

The sender (Alice) generates a string of random classical bits (0s and 1s). For each bit, she randomly picks one of three measurement bases:

  • Z basis (⊕) — Encodes 0 as |0⟩ and 1 as |1⟩ (standard basis)
  • X basis (⊗) — Encodes 0 as |+⟩ and 1 as |-⟩ (Hadamard basis)
  • Y basis (◉) — Encodes 0 as |+i⟩ and 1 as |-i⟩ (circular basis)

Each bit is encoded into a single qubit using its chosen basis. This creates a quantum circuit for each qubit, which is then sent over the quantum channel. Using 3 bases instead of 2 (like BB84) makes eavesdropping harder to hide.

2

📡 Quantum Channel Transmission

The prepared qubits travel from Alice to Bob through a quantum channel (e.g., a fiber-optic cable carrying single photons).

If Eve is enabled: She performs an intercept-resend attack. For each qubit, Eve:

  1. Intercepts the qubit in transit
  2. Randomly picks a basis and measures it (collapsing the quantum state)
  3. Prepares a new qubit based on her measurement result and sends it to Bob

Because Eve doesn't know Alice's basis, she picks the wrong one ~2/3 of the time. When she measures in the wrong basis, she gets a random result and destroys the original quantum information. This is the key insight — quantum mechanics forbids perfect cloning of unknown states (No-Cloning Theorem).

3

🟢 Bob Measures Qubits

Bob receives each qubit and, independently of Alice, randomly picks one of the three bases (Z, X, or Y) to measure it in.

If Bob picks the same basis Alice used → he gets Alice's exact bit (deterministic result).

If Bob picks a different basis → he gets a completely random result (50/50 chance), which is useless.

At this point, Bob has a list of measurement results, but doesn't yet know which ones are correct.

4

🔗 Basis Reconciliation (Sifting)

Alice and Bob publicly announce which basis they used for each qubit (but NOT the bit values). They keep only the bits where they happened to use the same basis and discard the rest.

With 3 bases, each qubit has a 1/3 chance of matching, so roughly 33% of qubits survive sifting. (BB84 with 2 bases has 50% survival.)

After sifting, Alice and Bob should have identical bit strings — unless the channel was noisy or someone was eavesdropping.

5

📊 QBER Estimation

Alice and Bob sacrifice a random subset (~50%) of their sifted bits and publicly compare them to estimate the Quantum Bit Error Rate (QBER).

No Eve, no noise → QBER ≈ 0% (bits match perfectly).

With Eve → QBER ≈ 33% for Six-State protocol. This is because Eve guesses the wrong basis 2/3 of the time, and when she does, she has a 50% chance of flipping the bit: 2/3 × 1/2 = 1/3 ≈ 33%.

The detection threshold is ~16.7% (half the theoretical Eve-QBER). If the measured QBER exceeds this, eavesdropping is suspected and the key is marked as compromised.

6

🔑 Shared Key Generation

The remaining sifted bits (those NOT used for QBER estimation) become the shared secret key. Both Alice and Bob now hold identical random bit strings, generated via quantum mechanics.

If QBER was below the threshold, the key is considered secure — no eavesdropper could have learned it without introducing detectable errors.

In a full implementation, additional steps like privacy amplification (shortening the key using hash functions) and information reconciliation (error correction) would be performed. This demo shows the core QKD process.

7

🔒 Message Encryption (OTP)

The message is encrypted using a One-Time Pad (OTP) — the only encryption scheme that is information-theoretically secure (proven unbreakable, regardless of computational power).

Each character is converted to 8-bit binary, then XOR'd bit-by-bit with the shared key:

ciphertext = message ⊕ key

The encrypted ciphertext (shown in hex) is meaningless without the key. OTP requires the key to be as long as the message and used only once — which is exactly what QKD provides: fresh random keys for every message.

8

🔓 Message Decryption

The receiver XOR's the ciphertext with their copy of the shared key to recover the original message:

message = ciphertext ⊕ key

If no Eve: Alice and Bob's keys match perfectly → message decrypts correctly ✅

If Eve was present: Eve's interception corrupted some key bits → decryption may produce garbled text, and the QBER will have flagged the compromise ⚠️
